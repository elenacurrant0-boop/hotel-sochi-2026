## 1. Вход и роли

При открытии приложения появляется экран ввода пароля.

| Роль | Что видит |
|------|-----------|
| **GUEST** | Основная, Детальный расчёт, Медцентр |
| **OWNER** | + Сезонные продукты, Партнёры и каналы |
| **ADMIN** | Всё выше + Пояснительная записка |

## 3. Вкладка «Основная»

### Что здесь

Главный дашборд модели: номерной фонд, загрузка, выручка, GOP, точка безубыточности.

### Алгоритм работы

**Шаг 1 — Номерной фонд**
- Указать количество номеров по категориям (стандарт, делюкс, люкс и т.д.)
- Итог по фонду считается автоматически

**Шаг 2 — Загрузка**
- Ввести плановую загрузку по месяцам (%) для каждой категории
- Глобальный коэффициент загрузки (`globalOccAdj`) — множитель на все месяцы сразу
- Помесячные коэффициенты гостей (`monthlyGuestCoeff`) — уточняют загрузку конкретного месяца
- Факт загрузки вводится отдельно в таблицу план/факт

**Шаг 3 — Пакеты питания**
- Миксы пакетов по месяцам (`pkgMixByMonth`) задают долю каждого пакета
- Сумма долей должна быть 100% — модель проверяет это автоматически

**Шаг 4 — Чтение результатов**
- Выручка = загрузка × тариф × дни × кол-во номеров
- GOP = Выручка − OPEX
- Точка безубыточности — при каком % загрузки GOP = 0

## 4. Вкладка «Детальный расчёт»

### Что здесь

Таблица ценовых периодов × пакеты. Показывает, какие цены действуют в каком месяце.

### Ценовые периоды (P0–P9)

| Индекс | Даты | Назначение |
|--------|------|------------|
| P0 | 01.01–31.01 | Низкий январь |
| P1 | 01.02–28.02 | Низкий февраль |
| P2 | 01.03–30.04 | Ранняя весна |
| P3 | 01.05–31.05 | Поздняя весна |
| P4 | 01.06–30.06 | Начало сезона |
| P5 | 01.07–31.08 | Высокий сезон |
| P6 | 01.09–30.09 | Бархатный |
| P7 | 01.10–31.10 | Осень |
| P8 | 01.11–31.12 | Поздний сезон |
| P9 | (внутренний) | Отображается для янв/фев с другими датами |

> **Важно:** P9 (01.11–28.12) используется в Детальном расчёте для января и февраля с корректными `displayDates`. Баг исправлен.

### Алгоритм работы с ценами

1. Выбрать период (строка таблицы)
2. Ввести цену за номер/ночь для каждого пакета
3. ПРОМО-тариф: см. отдельный раздел ниже

### ПРОМО-тариф — алгоритм

Каждый период имеет **имя** и **режим**:

**Режим «авто −X%»** (по умолчанию)
- Цена ПРОМО = базовый пакет × (1 − скидка%)
- Поле цены заблокировано (серое)
- Скидка и базовый пакет задаются в Настройках (панель справа)

**Режим «ручная цена»** (оранжевый бейдж)
- Цена вводится вручную — независимо от базового пакета
- Например: для периода P0 можно задать «ФСС» = 3 800 ₽/ночь

**Как переключить режим:**
1. В строке нужного периода в колонке ПРОМО — кликнуть на бейдж «авто −X%» или «ручная цена»
2. Режим переключится; при «ручная цена» поле разблокируется

**Как изменить название:**
1. Кликнуть на поле с надписью «ПРОМО» в шапке колонки нужного периода
2. Ввести своё название (например, «ФСС», «Акция март», «Корпоратив»)

## 5. Вкладка «Медцентр»

### Что здесь

Расчёт выручки медицинского центра по трём потокам гостей.

### Три потока

| Поток | Пакеты | Конверсия | Логика |
|-------|--------|-----------|--------|
| **Мед-гости** | MED | 50% | Уже приехали лечиться → покупают доп. процедуры |
| **Велнес-гости** | ULTRA, SPA | 20% | Едут за отдыхом → зайдут в МЦ если предложить |
| **Туристы** | BB, HB, FB, PROMO | 5% | Едут за морем → зайдут только если легко и дёшево |

### Алгоритм расчёта

```
Гостей в день (месяц) = кол-во номеров × загрузка% × гостей/номер
Поток в МЦ = Гостей в день × конверсия%
Выручка МЦ = Поток в МЦ × средний чек × дней в месяце
```

### Настройки МЦ (в правой панели)

- Конверсии по потокам — менять при изменении стратегии
- Средний чек по потокам — обновлять по факту
- Гостей на номер — обычно 1.6–1.8

## 6. Вкладка «Сезонные продукты»

### Что здесь

CRUD-карточки дополнительных продуктов (программы, акции, спецпредложения).

### Алгоритм добавления продукта

1. Нажать «Добавить продукт»
2. Заполнить форму:
   - **Название** — «Весенний SPA-уик-энд», «Детокс март» и т.д.
   - **Пакет-основа** — к какому базовому пакету привязан
   - **Даты действия** — с ... по ...
   - **Цена** — итоговая цена продукта
   - **Теги** — для категоризации
3. Нажать «Сохранить»

### Флаг «В записке»

- Если включён — продукт попадает в **Пояснительную записку** в раздел сезонных программ
- Если выключен — продукт хранится в базе, но в записку не попадает

### Бейдж «Сейчас активен»

- Появляется автоматически, если текущая дата попадает в диапазон дат продукта
- Менять вручную не нужно

### Хранение

Данные сохраняются в `localStorage` под ключом `sochi_seasonal_products`.
При очистке браузера — **данные потеряются**. Делать экспорт модели регулярно.

## 7. Вкладка «Партнёры и каналы»

### Что здесь

База каналов продаж и партнёров с расчётом эффективной цены по периодам.

### Типы партнёров

| Тип | Цвет | Описание |
|-----|------|----------|
| Корпоратив | синий | Скидка 10–20% или спецтариф на год |
| ТА | зелёный | Туристические агентства |
| OTA | оранжевый | Booking, Ostrovok и аналоги |
| ФСС / ДМС | фиолетовый | Фонды соцстрахования, ДМС |
| Акция | красный | Сезонные промоакции |

### Три типа ценообразования

**1. Скидка % от тарифа**
```
Цена партнёра = Базовый тариф × (1 − скидка%)
Пример: BB = 5 000 ₽, скидка 15% → партнёр платит 4 250 ₽
```

**2. Брутто + комиссия %**
```
Гость платит = Брутто (= открытый тариф)
Отель получает = Брутто × (1 − комиссия%)
Пример: BB = 5 000 ₽, комиссия OTA 18% → отель получает 4 100 ₽
```

**3. Нетто-цена (фикс)**
```
Фиксированная цена за номер/ночь независимо от сезона
Или задаётся помесячно (переопределение по периодам)
Пример: ФСС договорился на 3 800 ₽/ночь весь год
```

### Алгоритм добавления партнёра

1. Нажать «Добавить партнёра»
2. Указать:
   - **Название** партнёра или канала
   - **Тип** (корпоратив / ТА / OTA / ФСС / Акция)
   - **Базовый пакет** — от чего считается цена
   - **Тип ценообразования** — скидка / брутто+комиссия / нетто
   - **Значение по умолчанию** — % скидки, % комиссии, или нетто-цена
   - **Доля загрузки %** — сколько % от номерного фонда занимает этот канал
3. При необходимости — включить «Переопределить по периодам» и задать разные значения для каждого из 10 периодов
4. Сохранить

### Контроль долей

- Верхняя строка статистики показывает **суммарную долю** всех партнёров
- Если сумма > 100% — предупреждение
- «Прямые продажи» = 100% − сумма долей партнёров

### Хранение

Данные сохраняются в `localStorage` под ключом `sochi_partners`.

Значения берутся **прямо из модели** — автоматически:
- Выручка год

- Средняя загрузка

Кнопка «Печать / Сохранить PDF» в верхней части записки.
В браузере выбрать «Сохранить как PDF», масштаб 90%, книжная ориентация.

## 9. Сохранение и восстановление модели

### Экспорт (сохранить все настройки)

В панели настроек (шестерёнка) → кнопка **«Экспортировать»**:
- Скачивается JSON-файл со всеми параметрами модели
- Хранить в надёжном месте — это резервная копия

### Импорт (восстановить из файла)

В панели настроек → кнопка **«Импортировать»** → выбрать JSON-файл.
Все параметры восстановятся.

### Что сохраняется в localStorage

| Ключ | Данные |
|------|--------|
| `sochi_model_state` | Все параметры модели |
| `sochi_seasonal_products` | Сезонные продукты |
| `sochi_partners` | Партнёры и каналы |

> При очистке браузера (`Очистить данные сайта`) всё удаляется.
> **Делать экспорт JSON перед очисткой браузера!**

## 10. Ключевые алгоритмы расчётов

### Выручка номерного фонда

```
Выручка_месяц =
  Σ по категориям (
    кол-во номеров × загрузка% × дней_в_месяце ×
    Σ по пакетам (доля_пакета × цена_пакета_в_периоде)
  )
```

### GOP (Gross Operating Profit)

```
GOP = Выручка_номер + Выручка_МЦ + Выручка_прочее − OPEX
```

### OPEX (операционные расходы)

Делятся на:
- **Переменные** — зависят от загрузки (питание, уборка, расходники)
- **Постоянные** — не зависят от загрузки (зарплата, аренда, коммуналка)

### ПРОМО авто-расчёт

```
ПРОМО_цена[период] =
  если режим == 'auto':
    базовый_пакет[период] × (1 − скидка% / 100)
  если режим == 'manual':
    введённая вручную цена (не меняется автоматически)
```

### Эффективная цена партнёра

```
если тип == 'discount':
  цена = базовый_тариф × (1 − скидка% / 100)

если тип == 'commission':
  цена = базовый_тариф × (1 − комиссия% / 100)
  (гость платит полную цену, отель получает меньше)

если тип == 'netto':
  цена = фиксированная нетто-цена
  (или переопределение для конкретного периода)
```

### Переопределение по периодам (партнёры)

```
эффективное_значение[pIdx] =
  если задано periodValues[pIdx]:
    periodValues[pIdx]
  иначе:
    defaultValue
```

## Быстрый старт для нового сезона

1. **Обновить цены** → Детальный расчёт → ввести новые тарифы по периодам
2. **Проверить ПРОМО** → убедиться, что в нужных периодах правильный режим (авто/ручной)
3. **Обновить партнёров** → Партнёры и каналы → проверить доли и условия
4. **Обновить сезонные продукты** → добавить новые программы, отметить «В записке»
5. **Сгенерировать записку** → Пояснительная записка → проверить KPI → распечатать PDF
6. **Экспортировать модель** → сохранить JSON-резервную копию
