# CHANGELOG — Финмодель Отель Сочи 2026

---

## 2026-03-10

### Печать Прейскуранта
- Кнопка "Печать Прейскуранта" теперь открывает страницу в **альбомной ориентации** (A4 landscape)
- При печати прейскуранта скрываются: header ("Финмодель 2026") и блок KPI-карточек — выводится только сам прейскурант
- Шрифт цен в печатной форме — 11px, компактные отступы
- Классы для управления печатью: `pricelist-period-card`, `pricelist-table-wrap`, `printing-pricelist` (на body)

### Структура цен — 10 периодов
- Прейскурант переведён с 5 сезонов (объект) на **10 ценовых периодов** (массив `number[]` длиной 10)
- Добавлен `PRICE_PERIODS` — 10 записей с датами, `pIdx`, `sKey`, `isLow`
- `MONTHS.distribution` расширен: каждый элемент имеет и `pIdx` (ценовой период) и `sKey` (сезон загрузки)
- При загрузке из localStorage: **автомиграция** — если цены в старом формате объекта (5 ключей), сбрасываются на `initialPrices()` с утверждёнными ценами из прейскуранта

### Цены из утверждённого прейскуранта
- `initialPrices()` заполнена реальными данными из документа по 10 периодам × 3 типа номеров × пакеты BB/HB/FB/Ultra/SPA/MED
- ПРОМО рассчитывается как -10% от Ultra только в Низких сезонах (isLow = true)

## 2026-03 (ранее в сессии)

### Сегментация
- Добавлена вкладка **Сегментация** с помесячными долями сегментов (план/факт)
- Коэффициенты нетто по сегментам: Прямые 100%, ТО 78%, ФСС 70%, Корп 88%, OTA 85%
- Расчёт `mRev` через взвешенный коэффициент `segWeightedCoeff = Σ(share% × coeff%)`
- Таблица детальной проверки по сегментам: RN | ADR нетто | Выручка
- Вариант С: помесячная таблица с полной цепочкой расчёта (Комнаты × Дни → Загрузка → RN → ADR → Коэфф → Выручка)

### Сезоны и периоды
- Оставлено 5 сезонов (low/mid/high/peak/holidays) — **только для планирования загрузки**
- Добавлена помесячная таблица план/факт загрузки
